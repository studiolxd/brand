# Article (template)

Template reutilizable para páginas de artículo, caso de estudio o proyecto individual. A diferencia de `Pages/Home`, esta plantilla recibe todo su contenido mediante props y puede instanciarse para cualquier artículo del portfolio.

## Estructura de landmarks

- `<header>` — navegación principal (componente `Header`)
- `<main id="main-content">` — contenido del artículo; destino del skip link
- `<footer>` — pie de página (componente `Footer`)

El `id="main-content"` garantiza que el skip link del `Header` pueda llevar al usuario directamente al artículo, cumpliendo WCAG 2.4.1.

## Jerarquía de encabezados

Dentro de `<main>` la estructura es:

```
h1 — título del artículo (prop title)
├── h2 — primera sección de contenido (prop sections[0].title)
├── h2 — segunda sección
└── h2 — …
```

El `h1` es el título del artículo pasado mediante la prop `title`. Cada elemento de `sections` genera un `<section>` con su propio `h2`, formando una jerarquía plana y válida.

## Artículo

El contenido principal se renderiza dentro de un elemento `<article>` con las siguientes partes en orden:

1. `Tag` — categoría del artículo (prop `category`, coloreada con `tagVariant`)
2. `Heading level={1}` — título del artículo
3. Párrafo introductorio — resumen ejecutivo (prop `description`)
4. Imagen destacada — `aspect-ratio: 3/2`, ocupa el 100 % del ancho disponible; el `alt` se toma de `photoAlt` o, si no se proporciona, del `title`
5. Secciones de contenido — lista de `{ title, body }` pasada en `sections`

## Navegación configurable

Las props `navItems` y `featuredLink` permiten personalizar la navegación del `Header` sin necesidad de tocar el componente. Si no se pasan, se usan los valores por defecto exportados en `DEFAULT_NAV_ITEMS` y `DEFAULT_FEATURED_LINK`.
