# Botón

El botón es el elemento de acción principal del sistema. Existen cuatro variantes con roles distintos.

## Primary

Borde en color primario, fondo blanco y texto por defecto. En hover el fondo pasa a primario y el texto a inverso. Usado para acciones principales sobre fondos claros.

<Canvas of={ButtonStories.Primary} />

## Accent

Fondo en color accent-1 y texto oscuro. En hover el fondo pasa a accent-2. El texto es siempre oscuro para garantizar contraste suficiente.

> **Nota:** La prop `destructive` no tiene efecto sobre la variante `accent`. El color de fondo del accent es parte de su identidad visual y no es compatible semánticamente con la intención destructiva. Usa `variant="primary" destructive` o `variant="ghost" destructive` para acciones de alto riesgo.

<Canvas of={ButtonStories.Accent} />

## Outline

Borde y texto en color inverso sobre fondo transparente. En hover el fondo y el borde pasan a blanco y el texto a color por defecto. Para usar sobre fotografías o fondos oscuros.

<Canvas of={ButtonStories.OutlineOnDark} />

<Canvas of={ButtonStories.OutlineOnLight} />

## Ghost

Sin borde ni fondo. El texto toma el color por defecto de la superficie (oscuro sobre fondo claro, claro sobre fondo oscuro). En hover aparece subrayado. Para acciones secundarias de bajo peso visual, como cancelar, volver o acciones en línea dentro de un bloque de texto.

<Canvas of={ButtonStories.GhostOnLight} />

<Canvas of={ButtonStories.GhostOnDark} />

## Block

Cualquier variante admite el modificador `block` para ocupar todo el ancho del contenedor.

<Canvas of={ButtonStories.Block} />

## Tamaños

Tres tamaños disponibles: `sm`, `md` (por defecto) y `lg`. El tamaño se aplica mediante la prop `size`.

## Como enlace

### href

Cuando se pasa la prop `href`, el botón renderiza un `<a>` nativo. Apropiado para enlaces externos o cuando no se usa un router.

<Canvas of={ButtonStories.AsLink} />

El estado `disabled` en este caso añade `aria-disabled="true"` y elimina el `href`, ya que el atributo `disabled` nativo no existe en `<a>`.

<Canvas of={ButtonStories.AsLinkDisabled} />

### asChild (Next.js Link y otros routers)

Con la prop `asChild`, Button fusiona sus clases y event handlers directamente sobre el elemento hijo, sin añadir ningún elemento wrapper. Esto preserva la navegación client-side del router:

```tsx
import Link from 'next/link';
import { Button } from '@studiolxd/brand';

<Button asChild variant="primary">
  <Link href="/proyectos">Ver proyectos</Link>
</Button>
```

En Storybook se muestra con un `<a>` como stand-in del componente `Link` del router:

<Canvas of={ButtonStories.AsChild} />
